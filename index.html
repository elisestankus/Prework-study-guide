HTML

  • HTML makes up the structure of a webpage
  • An element is a type of content on a webpage
    • Elements are usually composed of an opening tag and a closing tag represented with angle brackets
  • HTML attributes define additional information about an element
    • HTML attributes are defined in the beginning element, not the closing element
  • The head element contains information (metadata) about the webpage.
    • Metadata contains information about accessibility, search engines, and performance
    • Metadata is not seen by users
  • Some common metadata tags found in the head element include the title, style, base, link, meta, and script elements
    • The title element defines the title of the webpage - used for Search Engine Optimization
    • The style element defines the CSS styles associated with the webpage
    • The base element defines the base URL for the webpage
    • The link element connects an external resource to the HTML document
    • The meta element defines metadata such as the character set, description, keywords, author, and viewport
    • The script element embeds client-side scripts in the HTML document
  • The body element represents the visible content shown to the user.
  • Common child elements within the body element include heading (h#), paragraph (p), unordered list (ul), ordered list (ol), list item (li), image (img), line break (br), and anchor (a) elements
  • Section elements are semantic elements that define a section in a document that contains thematic content. They allow us to organize content on our webpage.

CSS

  • Cascading Style Sheets (CSS) is a programming language used to define style rules for elements of an HTML page and format them.
    • CSS is used to define properties such as colors, aesthetics, visual layout, and organization
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
    • Inline CSS is used to style an element directly in the HTML file. Styles are applied using style attributes.
    • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file. A single change to a CSS rule will apply to all tagged page elements.
    • External CSS style sheet is the preferred way to style a webpage because it allows developers to keep all of their CSS rules in a separate file. A link element in the HTML's head is used to link the external CSS sheet to the HTML page.
  • A CSS rule consists of a selector and declarations
    • The selector defines the element or attributes to which the declarations will apply
    • Declarations contain two components: the CSS property we want to apply and the value of the property
  • An asterisk can be used as a selector to apply rules to all of the visible elements on the page
  • A CSS rule can be applied to multiple selectors by separating the selectors with a comma
  • CSS rules can be used to style text and images
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project
  • Git commands always start with "git"
  • git status: checks what branch we are currently on, shows modified files in working directory staged for your next commit
  • git add -A: adds all changes in the current working branch to the staging area
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot
  • git checkout -b branch-name: creates a new branch and switches to it
  • git push [alias] (branch): transmit local branch commits to the remote repository branch
  • git pull: fetch and merge any commits from the tracking remote branch
    • It is important to always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work

JavaScript

  • JavaScript is a dynamic programming language that allows users to interact with websites
  • Each line of JavaScript code ends with a semicolon
  • JavaScript is case sensitive
  • Variables can hold values that have different data types including strings, numbers, Booleans, arrays, and objects
    • String: sequence of text enclosed in single or double quote marks
    • Number: numbers don't have quotes around them
    • Boolean: true/false values, don't need quote marks
    • Array: structure that allows you to store multiple values in a single reference
    • Object: everything in JavaScript is an object and can be stored in a variable
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Conditonal statements and loops can be used to control the order in which code is executes, or if it is even executed at all
  • If statements: code will only execute if the statement meets a condition, or is true.
    • When a computer encounters an if statement in a JS file, it will check to see if the condition is truthy. If it is, then the computer will execute whatever action we ask it to do between the curly brackets
    • "else if" can be used if we have more than one condition
  • For loops use the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over
  • A for loop consists of three statements: the starting point, the condition, and the statement that allows the array to iterate over each item
  • A function is a set of instructions that tells the computer how to perform a certain task
    • Functions must be called by name in order to execute
    • A function's name is always followed by parentheses
    • The code to be executed is written inside curly brackets
